# AI 智能面试系统 — 项目设计文档

## 1. 系统架构

```
┌─────────────────────────────────────────────────────┐
│                    用户浏览器                         │
│              https://ai-interview.vercel.app         │
└────────────┬──────────────────────┬─────────────────┘
             │                      │
      ┌──────▼──────┐       ┌──────▼──────┐
      │   Vercel    │       │   Railway   │
      │  Nuxt SPA   │──/api→│  FastAPI    │
      │  (前端)     │       │  (后端)     │
      └─────────────┘       └──────┬──────┘
                                   │
                            ┌──────▼──────┐
                            │  Claude API │
                            │  (AI 引擎)  │
                            └─────────────┘
```

## 2. 技术选型

| 层 | 技术 | 选型理由 |
|---|---|---|
| 前端框架 | Vue 3 + Nuxt 3 | 生态丰富，SSR 可关闭，Vercel 原生支持 |
| UI 方案 | Tailwind CSS + 内联样式 | 快速开发，无需组件库 |
| 状态管理 | Pinia | Vue 3 官方推荐 |
| HTTP 请求 | Axios | 拦截器支持，自动附加角色参数 |
| 后端框架 | FastAPI | 异步支持，自动 OpenAPI 文档，Python AI 生态 |
| ORM | SQLAlchemy 2.0 | 成熟稳定，支持 SQLite/PostgreSQL |
| AI 引擎 | Claude API (Sonnet 4.6) | 多 Agent 架构，Tool Use 结构化输出 |
| 数据库 | SQLite（可切 PG） | 零配置开发，生产可迁移 |
| 测试 | pytest + pytest-asyncio | 17 个测试覆盖核心流程 |

## 3. 多 Agent AI 架构

```
面试流程:
                 ┌──────────────┐
   用户回答 ──→  │ Orchestrator │  ← 编排中心
                 └──┬───┬───┬──┘
                    │   │   │
         ┌──────────┘   │   └──────────┐
         ▼              ▼              ▼
   ┌──────────┐  ┌──────────┐  ┌──────────┐
   │ Scoring  │  │ FollowUp │  │ Question │
   │  Agent   │  │  Agent   │  │  Agent   │
   │ (评分)   │  │ (追问)   │  │ (出题)   │
   └──────────┘  └──────────┘  └──────────┘
                                    │
                               ┌────▼────┐
                               │ Report  │
                               │ Agent   │
                               │ (报告)  │
                               └─────────┘
```

每个 Agent 有独立的 System Prompt 和 temperature 参数，可单独调优。Orchestrator 负责状态管理和 Agent 调用顺序。

## 4. 数据库设计

### 核心表

```
jobs                    candidates
├── id (PK)             ├── id (PK)
├── title               ├── name
├── description         ├── email
├── requirements        ├── phone
├── source (hr/seeker)  ├── resume_text
└── created_at          └── created_at

interviews              messages
├── id (PK)             ├── id (PK)
├── job_id (FK)         ├── interview_id (FK)
├── candidate_id (FK)   ├── role (enum)
├── status              ├── content
├── overall_score(JSON) ├── scores (JSON)
└── timestamps          └── created_at

users                   invite_tokens
├── id (PK)             ├── id (PK)
├── username (unique)   ├── token (unique)
├── password_hash       ├── job_id (FK)
└── role                └── used
```

### 分析表

```
interview_datasets      prompt_versions
├── interview_id (FK)   ├── agent_name
├── question/answer     ├── version
├── topic               ├── system_prompt
└── scores (JSON)       └── is_active
```

## 5. API 设计

详见 `API契约文档.md`，核心端点：

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | /api/auth/register | 注册 |
| POST | /api/auth/login | 登录 |
| GET/POST | /api/jobs/ | 岗位管理 |
| POST | /api/interviews/chat-v2 | Agent 对话 |
| POST | /api/interviews/{id}/end-v2 | Agent 报告 |
| GET | /api/analytics/overview | 系统概览 |
| POST | /api/jobs/{id}/invite | 生成邀请链接 |
| POST | /api/invite/{token}/start | 候选人免登录面试 |

## 6. 前端路由设计

```
/login                    → 身份选择 + 登录/注册
/                         → 首页仪表盘（HR/求职者双视图）
/jobs                     → 岗位管理 / 目标岗位
/candidates               → 候选人管理（HR 专属）
/interviews               → 面试 / 练习记录列表
/interviews/create        → 开始模拟面试
/interviews/[id]          → 面试室（SSE 流式对话）
/reports/[id]             → 面试报告（SVG 图表）
/invite/[token]           → 候选人受邀面试（免登录）
/analytics                → AI 评估仪表盘
```

## 7. 安全设计

- 密码 SHA-256 + 随机盐哈希存储
- Cookie 存储角色和 Token（SameSite Lax）
- API 鉴权中间件（公开路径白名单）
- 求职者仅见自身数据，HR 仅见 HR 数据
- 候选人免登录面试通过一次性 Token 验证

# AI 智能面试系统 — 需求分析文档

## 1. 项目背景

技术岗位面试长期存在效率低、标准不统一的问题：HR 不懂技术无法判断候选人水平，面试官时间有限无法逐一深挖，求职者缺乏针对性练习渠道。AI 技术的成熟使自动化、标准化、个性化的智能面试成为可能。

## 2. 用户角色

| 角色 | 描述 | 核心诉求 |
|---|---|---|
| 求职者 | 准备技术面试的人 | 模拟真实面试、发现知识盲区、针对性提升 |
| HR / 面试官 | 负责招聘的管理者 | 批量筛选候选人、标准化评估、对比优秀人选 |

## 3. 功能需求

### 3.1 用户认证
- FR-01：用户通过选择身份（求职者/HR）后注册或登录
- FR-02：系统根据身份展示不同的功能和界面
- FR-03：支持用户名+密码注册，密码加密存储

### 3.2 求职者功能
- FR-04：设置目标岗位（粘贴 JD 或从已有岗位中选择）
- FR-05：上传个人简历（支持 PDF/Word/TXT 格式）
- FR-06：AI 模拟面试：根据岗位和简历动态生成技术问题
- FR-07：面试过程中 AI 实时追问和评分
- FR-08：面试结束后生成多维度评估报告（技术深度、逻辑、表达、实践）
- FR-09：查看历史练习记录和评分趋势

### 3.3 HR 功能
- FR-10：创建和管理面试岗位（含 JD 和技术要求）
- FR-11：生成候选人邀请链接（免注册，点击即面试）
- FR-12：查看候选人列表和面试记录
- FR-13：查看候选人面试评估报告
- FR-14：数据分析仪表盘（评分分布、高频话题）

### 3.4 AI 面试引擎
- FR-15：根据 JD + 简历动态生成技术题目
- FR-16：根据回答质量决定是否追问
- FR-17：4 维度评分（正确性、深度、逻辑、实践）
- FR-18：生成结构化综合评估报告

## 4. 非功能需求

| 需求 | 指标 |
|---|---|
| 响应时间 | 面试对话 < 10 秒 |
| 可用性 | 7×24 小时 |
| 安全性 | 密码加密、身份鉴权 |
| 可扩展性 | 支持 PostgreSQL 替换 SQLite |
| 兼容性 | PC + 移动端适配 |

## 5. 技术约束

- 必须使用 Claude API 作为 AI 引擎
- 数据库使用 SQLite（生产可切 PostgreSQL）
- 前端使用 Vue 3 + Nuxt 3
- 后端使用 Python FastAPI

## 6. 验收标准

- 求职者从注册到完成首场面试 < 3 分钟
- HR 生成邀请链接到候选人开始面试 < 1 分钟
- 面试报告生成时间 < 30 秒
- 评分维度完整率 100%（无缺失评分）
